关于模电笔记失踪

吐槽(借口)

之前这边模电笔记也是做到了十几份,但是突然电脑又又又又坏了😢

本来想着修好了再写,恰逢过年他店里不上班,没法送厂保修，

这里吐槽一句吧,戴尔的保修认真的吗,换主板换五次才勉强好，

现在虽然能用,但是十次开机九次BIOS提醒风扇未启动的,真是服了😓

你要不别保修了,直接让我重买一台得了.

正话

因为电脑坏了,当时的github仓库也是克隆在这台电脑上,重新部署?

重新部署是不可能重新部署的

(这玩意老出bug,刚刚我想起来写博客发现token过期了,重启配置一堆bug,最后回退重装了git2.38才解决)

这就是我断更的理由🐶

因为断更这几天我自己已经复习完了,不想再重新补一遍了,加上本人有强迫症,

不想一件事放着没做完(然后就直接把这件事从历史长河中删去🐶)

题外话

刚说到我刚才重新配置token遇到的bug,想起来之前挖坑说要讲一下自己这个博客部署的技术相关,

虽然暂时没啥时间系统地讲,但是这个东西就顺手讲了吧.

token获取

这次的问题是token过期,所以先重新去github官网申请一个token.

  1. 头像下拉列表settings

  2. 下拉左边developer settings

  3. 左边Personal access tokens下拉列表的Tokens(classic)

  4. generate new token下拉列表中generate new token (classic)

  5. Note文本框填写token的名字,想叫什么叫什么

    Expiration选择token有效期(我建议选下拉列表最下面的永久有效,虽然有安全问题,好在省事)

    Select scopes下面的框中勾选一级选项repo就行了

    填完就可以拉到最下面点绿色的按键generate token

    接下来转跳到界面显示一行乱码就是token,点后面的复制按钮复制(刷新后不再显示,建议马上复制)
